Question 90

Imagine you're a BI analyst working for a manufacturing company. You've created a Power BI dataset that analyzes production data stored in an on-premises SQL Server database. You want to publish this dataset to the Power BI service and schedule it to refresh daily so that your reports always have the latest production data.
Challenge: You know that the Power BI service cannot directly access on-premises data sources. You need to set up a secure connection between the Power BI service and your SQL Server database to enable data refresh.
Solution: Which four actions should you perform in sequence to configure this connection and schedule the daily refresh of your dataset?
Available Actions:
A. Configure an on-premises data gateway.
B. Add a data source to the gateway and specify the SQL Server database connection details.
C. Add the dataset owner's user account to the data source credentials in the gateway.
D. Configure a scheduled refresh for the dataset in the Power BI service.
E. Create a Power BI dataflow to connect to the SQL Server database.
F. Enable the "Enhanced data connectivity" option in the Power BI admin portal.

  • A. A, B, C, D
  • B. A, D, C, E
  • C. F, A, B, C
  • D. D, E, F, A
  • A. A, B, C, D
  • B. A, D, C, E
  • C. F, A, B, C
  • D. D, E, F, A

Explanation: To enable daily refresh of a dataset that connects to an on-premises SQL Server, you first need to install and configure an on-premises data gateway. Then, add the SQL Server database as a data source in the gateway and provide the connection details. Next, ensure that the dataset owner's credentials are configured for that data source. Finally, in the Power BI service, set up a scheduled refresh for the dataset. This sequence ensures secure access to the on-premises database and automated daily refreshes.
