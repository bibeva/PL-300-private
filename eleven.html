Question 110

Imagine you're a data analyst working for an online retail company. You're building a Power BI report to analyze customer orders and identify trends in purchasing behavior. You have access to an Azure SQL database that contains two tables: "Customers" and "Orders."
The "Customers" table contains information about each customer, such as their name, email address, and location. The "Orders" table contains details about each order, such as the order date, products purchased, and total amount. Both tables are linked by a common column called "CustomerID."
Challenge: You need to combine these two tables in Power Query Editor to create a single query that includes all the relevant information from both tables. This consolidated query will serve as the foundation for your Power BI report, allowing you to analyze customer orders in conjunction with customer demographics and purchasing history.
How should you combine the "Customers" and "Orders" tables in Power Query Editor to achieve this consolidated view?

  • A. Append the "Customers" query to the "Orders" query.
  • B. Merge the "Orders" query into the "Customers" query, using "CustomerID" as the join column.
  • C. Create a relationship between the "Customers" and "Orders" queries in the Power BI data model.
  • A. Append the "Customers" query to the "Orders" query.
  • B. Merge the "Orders" query into the "Customers" query, using "CustomerID" as the join column.
  • C. Create a relationship between the "Customers" and "Orders" queries in the Power BI data model.

Explanation: Merging the tables in Power Query allows you to create a single consolidated query containing both customer details and their corresponding orders. Using "CustomerID" as the join key ensures each order is correctly matched to the respective customer. Appending (Option A) would stack the tables vertically, which is incorrect for combining related data. Creating a relationship in the data model (Option C) keeps the tables separate and does not produce a single unified query.
